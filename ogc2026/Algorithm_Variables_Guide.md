# OGC 2026 Scheduling Algorithm: Variables Guide

이 문서는 `baseline_cpsat/myalgorithm.py`에서 사용되는 주요 변수명들의 역할과 의미를 설명합니다. 최신 버전에 맞게 전체 변수가 초기화 및 재작성되었습니다.

## 1. 전역 및 공통 데이터 변수 (Global & Common Data)
- **`prob_info`**: JSON에서 파싱된 전체 문제 데이터 딕셔너리. 블록 정보(`blocks`), 도면 정보(`bays`), 목적 함수 가중치(`weights`) 등을 포함.
- **`bays`**: 도면의 물리적 치수(너비, 높이)와 ID 정보를 담은 `Bay` 객체 리스트.
- **`timelimit`**: 알고리즘 실행에 허용된 최대 제한 시간 (초 단위).
- **`start_time`**: 알고리즘 구동을 시작한 시점의 시스템 타임스탬프.
- **`state` / `current_state` / `best_state` / `fixed_state`**: 특정 시점의 전체 블록 배치 상태를 저장하는 딕셔너리. 포맷은 `{block_id: (bay_id, x, y, orient_idx, entry, exit_t)}` 형태.
- **`raster_cache`**: 복잡한 블록의 폴리곤 형상을 고속 비트 연산을 위해 이진 마스크로 캐싱해두는 `RasterCache` 객체.

## 2. 블록 및 배치 관련 변수 (Block & Placement)
- **`b_id`**: 개별 블록의 고유 식별자 문자열 (예: `"block_1"`).
- **`b_info`**: 특정 블록의 요구사항 데이터(형상 정보, `release_time`, `due_date`, `processing_time`, `bay_preferences`, `workload` 등).
- **`bay_id`**: 도면(Bay)의 식별자(인덱스).
- **`x`, `y`**: 블록이 배치된 도면 상의 좌상단(바운딩 박스 기준) X, Y 좌표.
- **`o_idx` / `orient_idx`**: 블록의 회전 상태를 나타내는 인덱스 (0, 1, 2, 3 등).
- **`entry`, `exit_t` / `exit`**: 블록이 도면에 진입하여 작업을 시작하는 시간(`entry`)과 작업을 마치고 진출하는 시간(`exit_t`).
- **`bay_placed`**: 각 도면별로 이미 배치가 확정된 블록(`Block` 객체) 리스트를 저장하는 딕셔너리.
- **`bay_schedule`**: 각 도면별로 배치된 블록들의 진입/진출 시간 `(entry, exit_t)` 튜플을 저장하는 리스트 딕셔너리.

## 3. 비트마스크 그리드 변수 (RasterCache Variables)
- **`scale`**: 비트마스크 변환 시 1비트(픽셀)가 나타내는 실제 공간 길이 단위. (예: `scale=1`이면 1비트가 1m 공간을 의미)
- **`grid`**: 특정 도면의 일정 시간대에 장애물(이미 배치된 블록)들이 위치한 영역을 나타내는 정수형 이진 비트마스크 1차원 배열 (행 단위).
- **`gw`, `gh`**: 비트마스크 그리드의 전체 너비(가로 비트 수)와 높이(세로 행 수).
- **`b_mask`**: 새롭게 배치하려는 블록의 폴리곤 형상을 2D 비트마스크로 변환한 정수형 배열.
- **`bw`, `bh`**: 블록 비트마스크의 너비와 높이.
- **`blx0`, `bly0`**: 블록의 원본 다각형(Polygon) 바운딩 박스가 원점(0,0)을 기준으로 가지는 음수 오프셋. 실제 좌표계와 비트마스크 좌표계 간의 정렬 보정 시 사용.

## 4. ALNS (Adaptive Large Neighborhood Search) 변수
- **`U` / `removed`**: 파괴 연산(`destroy_*`)에 의해 현재 상태에서 스케줄이 제거되어 재배치가 필요한 블록 ID들의 집합.
- **`num_remove` / `curr_remove`**: 한 번의 파괴 연산 루프에서 도면에서 떼어낼 블록의 목표 개수.
- **`T` / `T_start` / `T_end`**: 시뮬레이티드 어닐링(Simulated Annealing) 과정의 온도 제어 변수. 높은 온도에서는 나쁜 결과도 확률적으로 수용함.
- **`stagnation`**: 새로운 최고점(`best_obj`)을 갱신하지 못하고 해가 정체된 루프 반복 횟수. (특정 수치 이상 시 `curr_remove` 규모를 키워 국소 최적해를 탈출)
- **`cands` / `candidates`**: `generate_candidates` 함수가 찾아낸, 특정 블록이 배치될 수 있는 여러 유효한 시공간 후보군(딕셔너리 리스트).
- **`conflicts`**: 재배치 시 고려되는 후보군 조합들 간에 물리적 위치나 시간, 또는 이동 경로(진입/진출)가 겹치는 충돌 쌍(Pair)들의 리스트.

## 5. 제약 만족 모델 변수 (CP-SAT Variables)
- **`model`**: OR-Tools의 제약 만족 문제 모델 인스턴스 (`cp_model.CpModel()`).
- **`solver`**: 설정된 모델을 기반으로 최적해를 연산하는 솔버 인스턴스 (`cp_model.CpSolver()`).
- **`X`**: 파괴된 블록들이 어느 후보군(`cand`) 위치를 선택할지 결정하는 불리언(0 또는 1) 의사결정 변수 딕셔너리. 키는 `(b_id, cand_idx)` 형태.

## 6. 목적 함수 및 멀티프로세싱 변수 (Objective & Multiprocessing)
- **`obj1`, `obj2`, `obj3`**: 전체 목적 함수(점수)를 구성하는 세 가지 요소. 
  - `obj1`: 납기 지연에 따른 페널티 합산
  - `obj2`: 도면(Bay) 간 작업량 불균형에 따른 페널티 합산
  - `obj3`: 블록이 선호하지 않는 도면에 배치되었을 때의 선호도 위반 페널티 합산
- **`workers_count`**: 멀티프로세싱을 위해 스폰(Spawn)되는 백그라운드 워커 프로세스의 개수.
- **`alns_duration`**: 각 독립 워커가 ALNS 메인 탐색 루프를 도는 데 허용된 순수 제한 시간.
- **`pool`, `async_results`**: 파이썬 `multiprocessing.Pool` 객체와 각 워커의 비동기 실행 결과를 추적하고 취합하는 변수.

## 7. 동시 퇴출 위상 정렬 변수 (Topological Sort)
- **`adj`**: 블록 간의 퇴출 간섭 관계(어떤 블록이 나가기 위해 어떤 블록이 먼저 치워져야 하는지)를 나타내는 방향 그래프 인접 리스트 딕셔너리.
- **`in_degree`**: 각 블록이 나가기 전에 먼저 나가야 하는(자신을 막고 있는) 다른 블록의 개수(진입 차수)를 추적하는 딕셔너리.
- **`queue`**: 진입 차수가 0이 되어 아무런 간섭 없이 즉시 나갈 수 있는 블록들을 담은 탐색용 대기열 리스트.
- **`topo_order`**: 위상 정렬을 통해 최종적으로 계산된, 충돌 및 간섭(S5 에러) 없이 안전하게 도면에서 빠져나갈 수 있는 블록들의 올바른 퇴출 순서 리스트.
